Creat your own chatbot.

product where you can provide your (or any) URL.
System would scrape and create chatbot from it.

Showcases:
  • From one parent URL going 2 levels deep to find all child url's.
  • Scraping the Child URL's (only scraping 100 URL's for demo).
  • Extracting the data like title, header, content.
  • Converting the content to embedding (openai embedding).
  • Storing the data + embedding values in Vector Database.
  • Based on Vector Database you can chat with your own data.

Chatbot with Openai or LLama2 Models with added functionality of speech support input.

You can chat with chatbot using text or speech. You have even functionality of selecting your model for chatbot. whether you need Openai GPT3 or Llama2 model.

Showcases:
  • Chat with your own data (from data scrap ID or general).
  • Selecting model to chat. Openai or Llama2 (Local).
  • Input of Voice Support. As an input of voice can also be provided. Internally Openai Whisper is used to convert speech to text.

Chatbot with Data Checks for better reliability.

In this chatbot there is an added layer of checks from user input whether the user is using harmful sentences or they needed human support (so we would redicrect to Human support and not waste their time chatting with chatbot).
For this we are using openai functions to perform user data checks.

Showcases:
  • Adding Openai Functions acting as user input validator to check user inputs and thereby make a decision.
  • As an application this helps to restrict harmful users to not use our bot (or to safeguard our model) and if the user really need human support then directly redirect them to one.
  • Through Openai Functions we can add other generative conditions based on user input and can get answer based on our functions.
